/api/chapters

GET

POST

Description: Returns the content of a specific chapter from a given novel.

Query Parameters

Parameter Type Required Description
n string Yes Novel name (with -chapters suffix, e.g., Shadow Slave-chapters)
c integer No Chapter number to retrieve

Example Requests

/api/chapters?n=Shadow%20Slave-chapters&c=1
/api/chapters?n=Shadow%20Slave-chapters

Sample Response

{
  "title": "Chapter 1: Nightmare Begins",
  "content": "It was late at night when Sunny opened his eyes..."
}

Error Responses

When something goes wrong, the API returns JSON formatted error messages. Common error responses include:

Chapter Not Found

{
  "error": "Chapter not found"
}

Novel Chapters Not Found

{
  "error": "No chapters found for novel: {novel_name}"
}

Quick Tips

  • Replace spaces in URLs with %20 or + (e.g., Shadow%20Slave-chapters).
  • Always include the -chapters suffix in the novel name.
  • Ensure chapter numbers are valid integers starting from 1.
